# Contributing to Awesome Aquarium Tech [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

Thank you for your interest in contributing to Awesome Aquarium Tech! This document provides guidelines for contributing to this curated list.

## How to Contribute

### Adding a New Project

First, **check if the project fits**: The project should be open source, related to aquarium technology (automation, monitoring, control), and actively maintained.

Next, **verify the project**: Ensure the project has a clear README or documentation, active repository (not archived), open source license, and some community activity or stars.

Then, **fork and clone** this repository.

**Add the project** to the appropriate category in `README.md` using the format `[Project Name](URL) - Description.` Include license information, platform details if applicable, list key features in bullet points, and keep descriptions concise (1-2 sentences).

**Sort alphabetically** within each category.

**Test your changes**: Ensure the links work and the formatting is correct.

Finally, **submit a pull request** with a clear description of the addition.

### Categories

Projects should be added to the most appropriate category:

- **Comprehensive Controllers**: Complete aquarium control systems with multiple features
- **Monitoring and Automation Systems**: Systems focused on monitoring and automation
- **Hardware Projects**: DIY hardware controllers, sensors, and physical devices
- **Mobile Applications**: Mobile apps for aquarium management and monitoring
- **Educational Resources**: Learning materials, tutorials, and guides
- **Specialized Tools**: Specific-purpose tools for particular aspects of aquarium care

### Improving Existing Entries

You can help improve existing entries by fixing broken links, updating descriptions if the project has changed significantly, adding missing information (license, features), or improving formatting or grammar.

### General Guidelines

Please be respectful and constructive in all interactions. Make one change per pull request for easier review, use descriptive commit messages, and follow the existing formatting style.

## Project Criteria

To be included in this list, projects should be open source with a clear license, be actively maintained (commits within the last year preferred), have a GitHub repository, be specifically related to aquarium technology, and have some documentation or usage instructions.

## Code of Conduct

Please be respectful and inclusive. Harassment or discriminatory behavior will not be tolerated.

## Questions

If you have questions about contributing, feel free to open an issue or contact the maintainers.
